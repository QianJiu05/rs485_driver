# 2026-03-23 寄存器映射封装总结

## 1. 变更目标
- 单独新增寄存器映射文件，集中管理 ModBus 操作寄存器。
- 让业务驱动改为引用寄存器映射，避免地址硬编码分散。

## 2. 新增文件

文件: Core/Inc/motor_registers.h

- 按 `#define XXX_REG` 风格封装寄存器地址。
- 覆盖输入寄存器区（5000~5020）与保持寄存器区（6000~6030）。
- 例如:
  - `#define FAULT_INFO_REG (5000U)`
  - `#define HEARTBEAT_REG (6000U)`
  - `#define CONTROL_MODE_REG (6001U)`
  - `#define TARGET_CURRENT_REG (6002U)`

## 3. 关联修改

文件: Core/Inc/motor_modbus.h

- 新增 `#include "motor_registers.h"`。
- 原有业务宏改为引用寄存器映射:
  - `MOTOR_MODBUS_REG_HEARTBEAT -> HEARTBEAT_REG`
  - `MOTOR_MODBUS_REG_CONTROL_MODE -> CONTROL_MODE_REG`
  - `MOTOR_MODBUS_REG_TARGET_CURRENT -> TARGET_CURRENT_REG`

## 4. 效果
- 后续新增控制功能（如转速/位置）时，仅需在 `motor_registers.h` 统一取址。
- 业务层代码可读性更高，避免“魔法数字”重复。
