# 电机速度闭环控制层搭建总结

## 日期
2026-03-24

## 变更概述
新建 motor_control 控制层，实现"以速度为闭环目标、以电流为输出"的 PID 控制架构，解耦控制逻辑与底层通信驱动。

## 架构层次
```
应用层 (main.c)
    ↓ motor_ctrl_init / motor_ctrl_loop
控制层 (motor_control.c)  ← 本次新建
    ↓ motor_modbus_xxx / pid_calculate
驱动层 (motor_modbus.c)
    ↓
协议层 (modbus_rtu.c)
```

## 新增文件

### Core/Inc/motor_control.h
- `motor_ctrl_t` 结构体：封装 PID 实例、运行状态、目标转速、实时转速、输出电流、故障码、通信错误计数
- `motor_ctrl_state_t` 枚举：STOPPED / RUNNING
- 可配置宏：`MOTOR_CTRL_MODBUS_TIMEOUT`（通信超时）、`MOTOR_CTRL_MAX_CURRENT_10MA`（电流限幅）

### Core/Src/motor_control.c
提供以下接口：

| 函数 | 功能 |
|---|---|
| `motor_ctrl_init(pid_cfg)` | 初始化 DWT 计时、Modbus 通信、创建速度环 PID 实例 |
| `motor_ctrl_start()` | 切换驱动器到电流模式，清空 PID 历史，进入 RUNNING 状态 |
| `motor_ctrl_stop()` | 写零电流停机，清空 PID，进入 STOPPED 状态 |
| `motor_ctrl_set_target_rpm(rpm)` | 设置目标转速 |
| `motor_ctrl_loop()` | 周期调用：心跳 → 读转速 → PID 计算 → 写电流 |
| `motor_ctrl_get_instance()` | 获取控制器实例只读指针（用于调试监测） |

### 控制流程
`motor_ctrl_loop()` 每次调用执行：
1. 心跳更新（维持驱动器通信）
2. 读取实时转速 (rpm)
3. 如果处于 RUNNING 状态，执行 PID 计算（measure=实时转速, ref=目标转速）
4. PID 输出限幅后写入目标电流 (10mA)

### 安全机制
- 通信失败时跳过本轮 PID 计算和电流写入，递增错误计数
- PID 输出经 `clamp_to_i16` 限幅到 ±`MOTOR_CTRL_MAX_CURRENT_10MA`
- 停止时主动写零电流并清空 PID 积分

## 修改文件

### Core/Src/main.c
- 头文件改为 `#include "motor_control.h"`
- 初始化改为 `motor_ctrl_init()` + `motor_ctrl_start()`
- 主循环简化为 `motor_ctrl_loop()` + `HAL_Delay(10)`
- PID 参数通过 `INIT_PID_CONFIG` 宏配置（Kp=10, Ki=1, Kd=0）

## 使用说明
1. 在 Keil 工程中添加 `Core/Src/motor_control.c` 和 `pid/pid.c`、`pid/drv_dwt.c` 到编译列表
2. 添加 `pid/` 到头文件搜索路径
3. 根据实际电机调整 `main.c` 中 `speed_pid_cfg` 的 PID 参数
4. `HAL_Delay(10)` 决定控制周期约 10ms + 通信耗时，按需调整
