# USART3 中断缺失导致 DMA 接收失败总结

## 日期
2026-03-24

## 问题现象
- DMA 发送数据到总线正常（示波器/逻辑分析仪可观测到波形）
- DMA 接收始终收不到从站反馈
- 改回轮询模式（`HAL_UART_Receive`）可以正常读取故障码等寄存器

## 根本原因
CubeMX 配置 USART3 DMA 时，遗漏了 **USART3 全局中断（USART3_IRQn）** 的使能。

STM32F4 HAL 库 DMA 发送完成的内部流程：
1. DMA 传输完成 → `UART_DMATransmitCplt` 回调
2. 该回调**并不直接调用** `HAL_UART_TxCpltCallback`，而是**启用 UART TC 中断**（`USART_CR1_TCIE`）
3. TC 中断触发 → `HAL_UART_IRQHandler` → `UART_EndTransmit_IT` → 设置 `gState = READY` → 调用 `HAL_UART_TxCpltCallback`

由于缺少 USART3 中断：
- TC 中断无法触发 → `HAL_UART_TxCpltCallback` 永远不被调用
- `modbus_uart_tx_done` 标志永远为 0
- `modbus_wait_dma_flag(&tx_done)` 每次超时返回 `MODBUS_ERR_TIMEOUT`
- `modbus_rtu_request` 在 `send_frame` 阶段就失败退出，**DMA RX 从未被启动**

这解释了"TX 数据确实发出，但 RX 收不到"的现象——不是 RX 硬件问题，而是软件流程在 TX 阶段就被超时中断了。

## 修复方案

### 1. stm32f4xx_hal_msp.c
在 USART3 MspInit 中添加 NVIC 中断使能：

```c
/* USART3 interrupt Init */
HAL_NVIC_SetPriority(USART3_IRQn, 0, 0);
HAL_NVIC_EnableIRQ(USART3_IRQn);
```

### 2. stm32f4xx_it.c
添加 USART3 中断服务函数：

```c
extern UART_HandleTypeDef huart3;

void USART3_IRQHandler(void)
{
  HAL_UART_IRQHandler(&huart3);
}
```

## 经验教训
- STM32 HAL DMA UART 需要同时配置 **DMA 通道中断** 和 **UART 全局中断**，缺一不可
- CubeMX 中启用 DMA 后，务必在 NVIC 页面确认对应 USARTx global interrupt 已勾选
- DMA TX "看起来正常发出"不代表软件流程正确完成——硬件级传输和 HAL 状态机是两回事
